# Как подключить дизайн-систему shadcnblocks к Claude Desktop (глобально, без проекта)

**Результат:** в любом чате Claude Desktop можно писать «покажи блоки из @shadcnblocks» и получать готовый код компонентов.

---

## Что нужно заранее
- Windows + установленный Claude Desktop.
- Node.js. Проверка: открыть PowerShell → ввести `node -v`. Если показывает версию — ок. Если нет — поставить LTS с nodejs.org.
- Действующий API-ключ shadcnblocks (платная подписка). Вид: `sk_live_...`.

---

## Шаг 1. Взять свежий API-ключ
shadcnblocks.com → Dashboard → **API Keys**. Проверить дату **Expires** — ключ не должен быть просрочен. Если старый истёк — удалить и создать новый. Скопировать целиком (он уже с `sk_live_`).

## Шаг 2. Создать папку-«хаб»
Любая пустая папка, например: `C:\Users\ВАШ_ЛОГИН\shadcn-hub`. Это не проект, просто место для двух файлов.

## Шаг 3. Файл `components.json` в папке
Имя строго `components.json` (не .txt). Содержимое:

```json
{
  "$schema": "https://ui.shadcn.com/schema.json",
  "style": "new-york",
  "tailwind": { "css": "app/globals.css", "baseColor": "neutral", "cssVariables": true },
  "aliases": { "components": "@/components", "utils": "@/lib/utils" },
  "registries": {
    "@shadcnblocks": {
      "url": "https://www.shadcnblocks.com/r/{name}",
      "headers": { "Authorization": "Bearer ${SHADCNBLOCKS_API_KEY}" }
    }
  }
}
```

`${SHADCNBLOCKS_API_KEY}` — это ССЫЛКА на ключ, а не сам ключ. Оставить дословно (с `$` и `{}`).

## Шаг 4. Файл `.env.local` в той же папке
Имя строго `.env.local` (с точкой в начале, не .txt). Одна строка:

```
SHADCNBLOCKS_API_KEY=sk_live_ВАШ_КЛЮЧ
```

Правила: без кавычек, без слова `Bearer`, без пробелов вокруг `=`, ключ один раз (он уже с `sk_live_`).

> Проводник Windows не даёт создать файл, начинающийся с точки.
> Обход: Блокнот → «Сохранить как» → имя в кавычках `".env.local"`, тип «Все файлы». Или создать в VS Code.

## Шаг 5. Открыть конфиг Claude Desktop
В приложении: **Settings → Developer → Edit Config**. Откроется `claude_desktop_config.json`.

## Шаг 6. Добавить коннектор
Внутри блока `"mcpServers"` добавить (путь заменить на свой):

```json
"shadcnblocks": {
  "command": "cmd",
  "args": ["/c", "cd /d C:\\Users\\ВАШ_ЛОГИН\\shadcn-hub && npx -y shadcn@latest mcp"]
}
```

Важно:
- Пути — с ДВОЙНЫМИ слэшами `\\`.
- Если серверов уже несколько — добавить `shadcnblocks` отдельной записью РЯДОМ (через запятую), не внутрь чужого сервера.
- Ключ в этот файл НЕ пишется — он берётся из `.env.local`.

Если серверов ещё нет вообще, весь файл целиком:

```json
{
  "mcpServers": {
    "shadcnblocks": {
      "command": "cmd",
      "args": ["/c", "cd /d C:\\Users\\ВАШ_ЛОГИН\\shadcn-hub && npx -y shadcn@latest mcp"]
    }
  }
}
```

## Шаг 7. Проверить JSON
Скопировать весь файл в jsonlint.com → должно быть **«Valid JSON»**. Одна лишняя запятая ломает ВСЕ коннекторы сразу.

## Шаг 8. Перезапустить Claude Desktop полностью
Не просто закрыть окно: трей (у часов) → **Quit**, или Диспетчер задач → снять задачу Claude. Затем открыть заново.

## Шаг 9. Проверить и пользоваться
В новом чате → индикатор MCP внизу поля ввода → `shadcnblocks` со статусом **connected**. Дальше писать:
- «покажи все блоки в @shadcnblocks»
- «покажи код @shadcnblocks/pricing3»
- «найди в @shadcnblocks hero с градиентом»

---

## Что способ умеет, а что нет
- ✅ Просмотр, поиск, чтение кода блоков — в любом чате, без проекта.
- ⚠️ Автоустановка (`add`) кладёт файлы в текущую папку (хаб), а не в ваше приложение. Чтобы вставить блок прямо в проект — работать через **Claude Code в папке проекта**. Хаб = витрина, а не установщик в чужой проект.

---

## Если статус «failed»
PowerShell:

```
cd C:\Users\ВАШ_ЛОГИН\shadcn-hub
npx -y shadcn@latest mcp
```

(В PowerShell пишите `cd` БЕЗ `/d`. Внутри конфига `/d` работает, потому что там запуск через `cmd`.)

- Запустился и «завис» в ожидании → сервер рабочий, дело было в перезапуске приложения (Ctrl+C для выхода).
- Ошибка про `components.json` → файл лежит не в той папке.
- Ошибка 401/403 → проверить ключ: `curl -H "Authorization: Bearer sk_live_..." https://www.shadcnblocks.com/r/hero2`. JSON = ключ рабочий; ошибка = ключ неверный/просрочен.

---

## Скрытые факторы (упускают почти все)
1. Одна опечатка в JSON = падают ВСЕ коннекторы. Всегда валидатор (шаг 7).
2. Тип файла важнее имени. `.env.local` и `components.json` часто сохраняются как `.txt` — включите показ расширений в проводнике и проверьте.
3. Просрочка ключа = тихая смерть. Коннектор не ругается заметно — просто перестаёт отдавать блоки (401). Запишите дату Expires.
4. `cd /d` (cmd) vs `cd` (PowerShell) — разный синтаксис. Люди путаются и думают, что всё сломано.
5. Ключ живёт в ОДНОМ месте: в `components.json` — только ссылка `${SHADCNBLOCKS_API_KEY}`, реальный ключ — только в `.env.local`.

---

## Альтернативные / «серые» варианты
- **Не держать ключ в `.env.local`:** вписать в блок `env` конфига → `"env": { "SHADCNBLOCKS_API_KEY": "sk_live_..." }`. Тогда `.env.local` не нужен, но ключ лежит в JSON.
- **CLI не подхватывает `.env.local`:** прогнать через dotenv-cli — в `args` хвост заменить на `... && npx -y dotenv-cli -e .env.local -- npx -y shadcn@latest mcp`.
- **Хостинговый коннектор «вставил URL — и всё»:** есть только у другого сервиса — shadcn.io (отдельная подписка ~$19/мес): Settings → Connectors → Add Custom Connector → их URL с токеном. Для shadcnblocks такого нет.
- **Без подписки:** ставить бесплатные блоки из публичных реестров (@shadcn и подобные). Платный каталог shadcnblocks так не откроется.
- **Не зависеть от ключа:** вытащить блоки один раз через `curl` с ключом и вшить код в репозиторий навсегда. На грани условий лицензии — с оглядкой.

---

## Безопасность
- Не публикуйте ключи в чатах, скриншотах, git. Если ключ куда-то попал открытым текстом — перевыпустите (rotate) его в дашборде.
- `.env.local` держите только локально; в репозиторий не коммитьте (добавьте в `.gitignore`).
